Deploying to WolfVox

WolfVox Cloud is built for instant deployments. We've removed the friction of Git pushes and SSH keys for a direct Drag-and-Drop workflow.

How to Upload

To launch a new project on the edge, follow these steps:

1. Navigate to 'New Project' from your Dashboard.
2. Select your local file (index.html, styles.css, etc).
3. Drag it directly into the upload zone.
4. Click [DEPLOY].

Once deployed, your files are synced to our global S3 buckets and served via our Render edge network.

How to Update

Updating your site is intelligent. The system detects updates based on Filename Consistency. You don't need to delete your old project to make a change.

Sync Rule: To update an existing file, drag in a new file with the exact same name as the hosted one.

Steps to update:

  • Modify your file locally (e.g., change text in index.html).
  • Drag the updated index.html into the project dashboard.
  • The system identifies the matching name and overwrites the old data in the Supabase bucket.
  • Click Deploy to purge the edge cache.

Routing & URLs

Every project is assigned a predictable, professional URL structure. This allows you to link assets between projects easily.

https://wolfcloud.onrender.com/:user/:repo

Replace :user with your unique WolfVox username and :repo with the name you gave your project during deployment.

Data Persistence

Unlike the WolfVox Forms platform (which clears data every 24 hours), WolfVox Cloud is designed for permanence.

Supabase Storage: All files uploaded to Cloud Hosting are stored in persistent S3-compatible buckets. Your site will remain online until you manually delete it or violate community terms.

Policy Enforcement

WolfVox maintains a professional environment. We utilize automated scanning for hosted content.

  • No Malware: Hosting malicious scripts results in an instant hardware ban.
  • No Phishing: Attempting to clone login pages will result in immediate repo deletion.
  • Verification: High-traffic repos may require a one-time Human Verification check.
